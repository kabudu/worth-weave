# Governance

Worthweave currently uses a maintainer-led governance model.

## Roles

- **Maintainer:** sets product direction, reviews changes, manages releases and security responses, and protects the project's financial-integrity and privacy invariants.
- **Contributor:** proposes issues or pull requests and participates in technical and product discussion.

## Decisions

Routine changes are decided through pull-request review. Material changes to financial methodology, database compatibility, broker schemas, privacy, security, licensing, or release infrastructure require explicit maintainer approval and corresponding documentation.

Consensus is preferred, but the maintainer makes the final decision when trade-offs remain unresolved. Decisions should be explained in the relevant issue or pull request.

## Releases and security

Only maintainers may create release tags or access Apple signing and notarisation credentials. Security reports follow [SECURITY.md](SECURITY.md) and may be handled privately until coordinated disclosure.

This model may evolve as the contributor community grows. Governance changes will be proposed and reviewed publicly.
